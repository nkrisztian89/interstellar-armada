Interstellar Armada Editor tutorial 03

Equipment and loadouts

Setting equipment using preset and custom loadouts for individual ships and squads

In this step-by-step walkthrough, we will update the mission we created in the first two tutorials, setting specific equipment for all the spacecrafts in multiple ways.

During the course of this tutorial, you will learn how to

  • set the equipment of a spacecraft or whole squad using a preset loadout
  • set different equipment loadouts for ships in the same squad
  • set a custom loadout by picking the pieces of equipment manually

This tutorial is made for the Interstellar Armada Editor version 0.5.4. We assume you have completed the second tutorial chapter and we will continue to edit the mission created there.

  1. Importing the mission

    Import the mission into the editor as you have learned to do in the previous tutorial.

  2. Setting a loadout for a single spacecraft

    Spacecrafts can have various different pieces of equipment, such as sensors, weapons, missiles, shields, propulsion systems and jump engines. To make managing these easier while editing a mission, nearly all spacecraft classes come with a set of predefined equipment loadouts. To view them, open the spacecraftClasses list in the left panel (within the classes section) and click a spacecraft class, e.g. wolf. In the right side properties panel you will see a loadouts property, and by opening it you can examine the details of all the loadouts of this class.

    Note that although you can add new loadouts to a spacecraft class with the editor, these cannot be used in missions as they are not exported with the mission file! This feature is planned for later. For now, stick to using the built-in loadouts! The same goes for new classes or resources created in the editor, they will not be exported with the mission and thus can't be used yet.

    Hint: when viewing a spacecraft class, you can select any of its loadouts at the top options panel, and you will see a summary of the important stats such as speed, firepower and shield capacity with that loadout equipped at the bottom of the window in the info panel. This way you can quickly gauge the strength of a loadout without having the examine the details of the equipment it includes.

    The simplest way to define equipment for a spacecraft is by using one of these loadouts. Select the "Raider 4" fighter. You will see its loadout property is listed as "pirate-full". This is because each class can have a default loadout specified, which is applied if no loadout is set for it explicitly. Now let's make this fighter more powerful. Set the loadout property to "pirate-elite", which includes a strong shield next to the 3 guns that pirate-full has.

    At this point you have set up Raider 4 to be more powerful than the other 3 members of its squad.

  3. Setting a loadout for a whole squad

    Since squads (or multiple spacecrafts defined by the same spacecraft entry) share the same spacecraft class, you can easily set a shared loadout that will apply to all of them.

    Let's go ahead and select the raider squad by clicking any of its first 3 members in the preview or selecting the "raider (3)" entry in the spacecrafts list.

    You will see both a loadout and a loadouts (plural) property available. Set the singular loadout to "pirate-medium".

    This will set the loadout of all the 3 members defined by this entry to the selected loadout, so in this case they will all become a bit weaker, having only 2 guns instead of 3.

  4. Setting different loadouts for members of the same squad

    As you have noticed, in case of multiple spacecrafts being defined, there is also a plural loadouts property available. We can use this to choose different loadouts within the same spacecraft group or squad.

    Select the player's squad by clicking on any of its members or selecting it in the spacecrafts list.

    Set the loadouts property, which will initially become and empty list. Open it and add 3 elements to the list by clicking +. Now you can select the 3 different loadouts you wish the 3 squad members to use. Select "player-tier1" for the first and "imperial-tier1" for the other two elements.

    Now you have made the player use a player specific loadout while the wingmen will use the regular imperial version.

  5. Setting specific equipment for a spacecraft

    Maybe we have made Raider 4 a little bit too strong. We want it to have 2 guns equipped but still have the shield. Unfortunately, there is no such loadout available for us to choose, so we will have to define this equipment piece by piece.

    Select Raider 4 and unset the loadout property (by clicking the x button appearing next to it), which you have previously set to "pirate-elite". This will unlock the equipment property, which was previously hidden.

    Note that the editor doesn't let you set properties that conflict with each other. Whenever you set a property, the others which would conflict with it are automatically hidden. Therefore if you don't see a property that you want to set, it means you have some other property values you need to change or unset to make it available again.

    Set the equipment property and open it (by clicking the Loadout button that appears).

    Set the basedOn property of the equipment to "pirate-medium". This way we only need to set the differences in equipment compared to that loadout, and don't have to set up everything from scratch.

    The pirate-medium loadout already has the weapons, missiles, sensors and jump engine set up as we want it. So let's just set the propulsion property to "pirateFighterMk2" and the shield property also to "pirateFighterMk2".

    Now you have a custom equipment set up for Raider 4.

  6. Exporting the mission

    This concludes the third tutorial! At this point, you have full control over the equipment of all the spacecrafts in the mission.

    Export and load the mission as explained in the first tutorial.

    Using the custom equipment feature you can set up interesting and unexpected equipment combinations for spacecrafts! Feel free to experiment, but note that in future game versions some variations will not work, as each equipment and slot will have more detailed types that need to match. (such as not being able to equip huge turrets or cannons or large amounts of missiles on small fighters)
